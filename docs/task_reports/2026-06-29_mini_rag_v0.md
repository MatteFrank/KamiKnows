Date: 2026-06-29

## Task Objective

Build the first minimal mini-RAG v0 pipeline over the Fase 1G RAG-ready FastCaloSimulation dataset.

Pipeline scope:

```text
question
-> local retrieval over chunks
-> optional Ollama/Qwen generation
-> grounded prompt with citation instructions
-> retrieved context and answer artifacts
-> manual review checklist
```

This task did not add fine-tuning, LoRA, QLoRA, training, discovery generation, a knowledge graph, figure extraction, FAISS, Chroma, vector DB, or corpus scaling.

## What This Demonstrates

- The Fase 1G dataset can drive local retrieval.
- A deterministic stdlib TF-IDF retriever can return top-k chunks with scores.
- Retrieval outputs are traceable to `chunk_id` and `arxiv_id`.
- Ollama/Qwen generation can run over retrieved contexts.
- Backend errors are handled gracefully if they occur.
- Manual review scaffolding is produced for grounding and citation checks.

## What This Does Not Demonstrate

- Scientific correctness.
- Reliable citation correctness.
- Robust RAG quality.
- Semantic retrieval quality.
- Production QA behavior.
- Discovery generation.
- Fine-tuning or model improvement.

## Recommended Next Step

Manually review `outputs/rag_v0_fastcalosim/rag_manual_review_checklist.md`, focusing first on the three retrieval misses and on whether generated answers cite explicit `chunk_id` values. After review, decide whether the next micro-task should be prompt/citation formatting, query rewriting, or a slightly stronger retriever.
